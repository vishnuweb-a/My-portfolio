# Vishnu Bhardwaj Portfolio 3.0

## Design Philosophy

This portfolio is **not** a template portfolio.

It should feel like a professionally designed software product rather than a personal website.

The design language should communicate:

* Engineering
* Precision
* Professionalism
* Confidence
* Simplicity
* Technical depth

Every component should have a purpose.

Nothing should exist purely for decoration.

Avoid flashy animations that reduce usability.

The design should impress recruiters, engineering managers, startup founders, and freelance clients.

# Coding Standards for the AI Agent

The generated UI must look handcrafted by an experienced product designer—not AI-generated.

Do NOT create "vibe coding" interfaces.

Avoid generic layouts that resemble common AI-generated portfolios.

Do NOT overuse gradients, glowing borders, glassmorphism, floating shapes, oversized hero sections, or unnecessary animations.

Maintain consistency in spacing, typography, color, and interaction patterns.

Use only relevant, real-world icons from approved icon libraries. Never substitute unrelated icons for aesthetic reasons.

Every visual element must have a clear functional purpose.

Favor simplicity, clarity, and usability over visual spectacle.

Write clean, modular, reusable React components with a scalable folder structure and consistent naming conventions.

Design every page to production quality rather than as a prototype.

The final experience should resemble a polished product from companies like Linear, Vercel, GitHub, or Notion, reflecting strong engineering discipline and thoughtful product design rather than an AI-generated website.
